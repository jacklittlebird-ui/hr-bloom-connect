خطة من 5 مراحل لإدارة السيارات. سأنفذها مرحلة-مرحلة بعد موافقتك، وسأخبرك عند نهاية كل مرحلة. الكود الموجود حالياً يحتوي على أساس جيد سنبني فوقه.

---

### 1) أذونات حسب الدور لكل تبويب
الحالة الراهنة: `StationManagerPortal` يمرر `allowedStationIds` لكل تبويب من تبويبات السيارات الخمسة، ومكوّنات `VehicleRegistry / VehicleLicenseTracking / VehicleMaintenance / FleetByStation / LicenseAlerts` كلها تستقبل الـ prop وتفلتر السيارات/التراخيص/سجلات الصيانة بناءً عليه. RLS على جدول `vehicles` و `vehicle_maintenance` يقيّد القراءة بنفس النطاق لكل دور.

ما سأضيفه:
- التحقق من أن **كل** تبويب يطبق الفلترة فعلياً (مراجعة `FleetByStation` و `LicenseAlerts` خصوصاً) وإصلاح أي تسرّب.
- منع ظهور حقل اختيار "كل المحطات" في الفلاتر للأدوار المحدودة (تثبيت النطاق على محطات المستخدم فقط).

---

### 2) تنبيهات تلقائية لاقتراب انتهاء التراخيص + إشعارات داخلية
- **Edge Function** جديدة `check-vehicle-licenses` تشتغل يومياً عبر `pg_cron` تفحص الـ 3 أنواع تراخيص (رئيسي / ستائر / نقل بري) وتنشئ سجلات في جدول `notifications` لكل من:
  - admin + hr (كل الأسطول)
  - area_manager / station_manager / station_hr (سيارات محطاتهم فقط)
- العتبات: منتهي / ≤7 يوم / ≤30 يوم.
- إضافة **فلتر حالة الترخيص** (الكل / منتهي / قريب / صالح) لتبويب `License Tracking` و `License Alerts`.

---

### 3) تحسين البحث وحفظ الفلاتر
لتبويبات Registry + License Tracking + Maintenance:
- توسيع البحث ليشمل: رقم اللوحة، اسم السائق، الكود، الماركة، الموديل، مقدم الخدمة.
- إضافة **نطاق تواريخ** (من/إلى) لتبويبَي License Tracking (تاريخ انتهاء) و Maintenance (موجود مسبقاً، سيتم توحيده).
- **حفظ الفلاتر** عبر `usePersistedState` بمفتاح فريد لكل تبويب، مع زر "إعادة ضبط".

---

### 4) إجراءات صيانة مجمّعة
في تبويب `Maintenance`:
- زر "إضافة صيانة لعدة سيارات".
- Dialog فيه: قائمة Checkbox للسيارات (مفلترة حسب نطاق المستخدم/المحطة)، نوع الصيانة، التاريخ، تاريخ الاستحقاق القادم، الحالة، التكلفة المشتركة، الوصف.
- إنشاء سجل صيانة لكل سيارة محددة دفعة واحدة (batch insert).
- بعد الحفظ: رسالة نجاح "تم إنشاء N سجل صيانة".

---

### 5) تصدير Excel / CSV / PDF لتقرير `FleetByStation`
- إضافة قائمة منسدلة "تصدير" بثلاثة خيارات تستخدم نفس مكتبات التصدير المعتمدة (`exportVehiclePdf` / `xlsx` / CSV مع BOM).
- التصدير يحترم الفلاتر النشطة (المحطة، نطاق البحث).
- زر التصدير يعرض **عدد الصفوف الحالية** (مثلاً: `تصدير (٤٢)`).
- تنسيق RTL كامل: ترويسة (عنوان، تاريخ التوليد، اسم المستخدم، المحطة، عدد السجلات)، أعمدة بترتيب عربي، تذييل بأرقام الصفحات في PDF، وتجميد الرأس + autofilter في Excel — نفس نمط `leavesExport.ts`.

### Database & Backend
- Edge Function جديدة فقط (`check-vehicle-licenses`).
- جدولة cron يومياً 06:00 بتوقيت القاهرة.
- لا حاجة لتغييرات schema (نستخدم `notifications` الموجود).

### تأثير الأداء
- جلب السيارات/الصيانة يستمر كما هو (محدود بـ scope).
- التنبيهات اليومية: استعلام واحد + إدراج دفعي.
